## 0. 데모 목적

이 데모의 목적은 AI법친의 핵심 가치를 가장 짧고 명확하게 보여주는 것이다.

이번 데모에서는 아래 3가지를 반드시 보여준다.

1. 사건이 단순 입력이 아니라 **실제 흐름으로 관리**된다는 점
2. AI 인터뷰 결과가 **사건 요약과 문서 생성**으로 이어진다는 점
3. 문서 승인 이후 **잠금·검증·전달 준비**까지 실무 흐름이 닫힌다는 점

---

## 1. 발표용 한 줄

AI법친은 사건 입력, AI 인터뷰, 사건 요약, 문서 생성, 승인·잠금·전달 준비까지 하나의 실무 흐름으로 이어지는 법률 업무 지원 플랫폼입니다.

---

## 2. 데모 기본 원칙

- 기능을 많이 보여주기보다 **흐름이 끊기지 않게** 보여준다.
- 클릭은 적게, 의미 설명은 짧게 한다.
- 세부 판정보다 **사용자 경험과 실무 연결**을 강조한다.
- 오류 가능성이 있는 경로는 피하고, 미리 준비된 대표 사건 기준으로 시연한다.

---

## 3. 데모 준비물

- [ ] 로그인 가능한 관리자 또는 변호사 계정
- [ ] 사건 상세까지 이미 생성된 대표 사건 1건
- [ ] 인터뷰 완료 상태가 반영된 사건
- [ ] 사건 요약 생성이 가능한 상태
- [ ] 문서 초안 생성 가능한 상태
- [ ] 승인 가능한 문서 1건
- [ ] 잠금 가능한 승인본 1건 또는 잠금 직전 상태
- [ ] 전달 준비 단계까지 보여줄 수 있는 선택 문서 1건

---

## 4. 권장 데모 순서

### Step 1. 로그인 후 사건 상세 진입

**보여줄 것**

- 로그인 후 사건 상세 화면 진입
- 상태 카드
- 진행 액션
- 문서 목록
- 타임라인

**말할 것**

- “AI법친은 사건 하나를 중심으로 인터뷰, 문서, 승인, 타임라인이 모두 연결됩니다.”

**체크**

- [ ] 대표 사건 상세가 즉시 열림
- [ ] 상태 카드와 액션 영역이 보임

---

### Step 2. AI 인터뷰 흐름 설명

**보여줄 것**

- 사건 상세에서 `AI 인터뷰` 진입
- 질문 진행 방식
- 저장/완료 가능한 구조
- 다시 사건 상세로 복귀

**말할 것**

- “질문셋은 사건에 맞게 진행되고, 답변은 저장되며, 완료 후 다음 단계로 자연스럽게 이어집니다.”

**체크**

- [ ] 인터뷰 화면 진입 가능
- [ ] 질문/진행률/저장 UI 확인 가능
- [ ] 사건 상세로 복귀 가능

---

### Step 3. 사건 요약 생성

**보여줄 것**

- 사건 상세의 `사건 요약(인터뷰 기반)` 패널
- `1차 요약 불러오기` 또는 `요약 미리보기`
- 생성 시각, 사건 상태, 개요/쟁점/체크리스트/고지문

**말할 것**

- “인터뷰 결과는 바로 사건 요약으로 정리되고, 참고용 자동 생성 결과라는 점도 서버에서 강제합니다.”

**체크**

- [ ] 요약 패널 노출
- [ ] 버튼 클릭 시 요약 표시
- [ ] 고지문 표시 확인

---

### Step 4. 문서 생성

**보여줄 것**

- `문서 생성` 버튼
- 문서 종류 선택
- 질문셋/템플릿 코드 표시
- 생성 후 문서 목록 반영

**말할 것**

- “인터뷰가 끝나면 문서 초안을 바로 생성할 수 있고, 질문셋과 템플릿 기준으로 구조화됩니다.”

**체크**

- [ ] 문서 생성 모달 열림
- [ ] 생성 후 문서 목록 갱신
- [ ] 선택 문서가 상세에 반영됨

---

### Step 5. 문서 검토와 문단 구조

**보여줄 것**

- 문서 검토 패널
- 문서 본문 미리보기
- 문단 구조 패널

**말할 것**

- “생성된 문서는 바로 검토하고, 문단 단위로 구조를 보면서 다듬을 수 있습니다.”

**체크**

- [ ] 검토 패널 표시
- [ ] 본문 미리보기 표시
- [ ] 문단 구조 패널 표시

---

### Step 6. 승인

**보여줄 것**

- `문서 승인` 버튼
- 승인 후 상태 변화
- 승인일/상태 반영

**말할 것**

- “승인 단계에서는 문서 승인과 사건 승인 단계가 함께 정합되도록 설계되어 있습니다.”

**체크**

- [ ] 승인 버튼 노출
- [ ] 승인 성공 후 화면 갱신
- [ ] 승인 상태 확인

---

### Step 7. 잠금

**보여줄 것**

- `승인본 잠금`
- 잠금 후 읽기 전용 전환
- 잠금일 표시
- 검증 페이지 링크 안내

**말할 것**

- “승인 후에는 승인본을 잠가서 고정하고, 이후 검증과 전달 준비 단계로 넘어갑니다.”

**체크**

- [ ] 잠금 버튼 노출
- [ ] 잠금 후 상태 `LOCKED`
- [ ] 검증 링크 확인

---

### Step 8. 전달 준비

**보여줄 것**

- 잠금된 문서 선택 시 `전달 완료 처리` 액션이 열리는 모습
- 상단 진행 액션과 패널 안내의 연결
- 타임라인/후속 운영 흐름

**말할 것**

- “잠금된 문서에서만 전달 준비가 열리고, 이력과 후속 처리 흐름도 남습니다.”

**체크**

- [ ] `DELIVER_DOCUMENT`가 잠금된 문서에서만 열림
- [ ] 후속 안내 문구 확인 가능

---

## 5. 발표 중 꼭 강조할 포인트

### 강조 1

AI법친은 단순 챗봇이 아니라 **사건 중심 워크플로**다.

### 강조 2

AI 인터뷰 결과가 **요약과 문서 생성**으로 실제 연결된다.

### 강조 3

문서 승인 이후 **잠금·검증·전달 준비**까지 실무 단계가 이어진다.

### 강조 4

핵심 축은 기능 존재가 아니라 **상태·권한·이력까지 닫힌 구조**다.

---

## 6. 피해야 할 시연 방식

- [ ] 처음부터 너무 많은 메뉴를 열기
- [ ] 아직 설명하지 않은 예외 흐름을 즉흥적으로 건드리기
- [ ] 관리자/의뢰인/변호사 역할을 한 데 섞어 보여주기
- [ ] 문서가 없는 사건, 인터뷰가 비어 있는 사건으로 시작하기
- [ ] 실패 케이스를 메인 시연으로 삼기

---

## 7. 발표자용 짧은 멘트 예시

### 시작

“AI법친은 사건을 만들고, AI 인터뷰로 정리하고, 요약과 문서 생성, 승인과 잠금까지 하나의 흐름으로 잇는 서비스입니다.”

### 인터뷰

“답변은 저장되고, 질문은 조건에 따라 바뀌며, 완료 후 다음 단계로 넘어갑니다.”

### 요약

“이 요약은 인터뷰 답변을 기준으로 자동 생성되며, 참고용이라는 점이 분명히 표시됩니다.”

### 문서 생성

“문서 초안은 질문셋과 템플릿 기준으로 만들어지고, 생성 후 바로 검토로 이어집니다.”

### 승인/잠금

“승인 이후에는 승인본을 잠그고, 검증과 전달 준비까지 실제 실무 순서로 이어집니다.”

### 마무리

“즉 AI법친은 단발성 AI 응답이 아니라, 사건을 실제 업무 흐름으로 마감할 수 있게 돕는 플랫폼입니다.”

---

## 8. 데모 실패 시 최소 복구 순서

1. 사건 상세로 돌아간다
2. 상태 카드와 문서 목록만 다시 보여준다
3. 이미 생성된 요약 또는 문서를 중심으로 설명한다
4. 승인/잠금 단계로 바로 넘어간다
5. “핵심은 흐름 연결”이라는 메시지로 정리한다

---

## 9. 공표 직전 최종 한 줄

AI법친 데모는 기능을 많이 보여주는 것이 아니라, 사건 상세 → AI 인터뷰 → 사건 요약 → 문서 생성 → 승인 → 잠금 → 전달 준비가 끊기지 않는 하나의 실무 흐름임을 보여주는 데 집중한다.

**이 문서의 용도:** 발표자용 **진행표** · **체크리스트** · **멘트 초안** · **실패 시 복구 플랜**을 **한 장**에 둔다.
