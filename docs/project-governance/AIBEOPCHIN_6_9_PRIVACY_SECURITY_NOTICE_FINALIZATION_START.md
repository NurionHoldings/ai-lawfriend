## 1. 문서 목적

이 문서는 AI법친 6.3~6.8에서 구현·검증한 사건 패키지 공유 기능에 대해,
개인정보·보안·동의문구를 최종 정리하기 위한 6.9 착수 기준을 고정한다.

이번 단계의 목적은 기능 추가가 아니라,
이미 동작하는 공유/조회/다운로드/요약본 출력 흐름에 어떤 고지문과 책임 경계를 붙일지 기준을 수렴하는 것이다.

핵심은 아래 세 가지다.

- 사용자가 무엇에 동의하는지 명확히 보이게 한다.
- 변호사·의뢰인·내부 운영자에게 보이는 책임 경계를 서로 모순 없이 맞춘다.
- 7.0 문서 작성/전자소송 제출 보조 패키지 기획으로 넘어가기 전에 법적 오인 가능성을 먼저 줄인다.

## 3. 정리 원칙

- 고지문은 UI 장식이 아니라 책임 경계와 오인 방지를 위한 정책으로 취급한다.
- 같은 행위를 설명하는 문구는 owner 화면, lawyer 화면, 출력물에서 서로 모순되면 안 된다.
- 공유 동의, 다운로드 허용, 로그 보관, 접근 차단은 각각 별도 의미를 가지므로 문장을 섞어 쓰지 않는다.
- AI 생성 결과는 보조 초안이며 최종 법률 판단/제출 책임을 대체하지 않는다는 원칙을 유지한다.
- 개인정보 제3자 제공 또는 열람 동의는 `공유 범위`와 `동의 문구` 양쪽에서 같은 의미로 보이게 한다.
- 전자소송 제출 보조 패키지는 제출 자동화가 아니라 준비 보조라는 한계를 명시한다.

## 4. 6.9 핵심 범위

6.9에서 최종 정리할 범위는 아래 10개로 고정한다.

1. 사건 패키지 공유 동의문구
2. 변호사 열람 고지문
3. 첨부파일 다운로드 고지문
4. 사건 패키지 요약본 출력 고지문
5. 공유 취소 후 접근 제한 문구
6. 접근 로그 / 다운로드 로그 보관 고지
7. AI 생성 문서의 법률 책임 고지
8. 변호사 최종 검토·승인 책임 문구
9. 개인정보 제3자 제공 / 열람 동의 구조
10. 전자소송 제출 보조 패키지 관련 책임 제한 문구

## 6. 이번 단계에서 포함하는 것

- 현행 사건 패키지 공유 흐름에 필요한 고지문 문구 정합성 정리
- 동의문구/면책문구의 공통 원칙 정리
- UI/API/출력물별 문구 차이 정리
- 개인정보 제3자 제공/열람 동의 구조 문서화
- 7.0 진입 전 책임 제한 문구 초석 정리

## 7. 이번 단계에서 하지 않는 것

- 신규 Prisma model 추가
- 공유 scope 필드 추가
- 실제 전자소송 제출 기능 구현
- 제출 자동화/서명 자동화 기능 구현
- 접근 로그 보관 주기 시스템 구현
- 문서 자동 승인 기능 구현
- 변호사 수임/상담 계약 기능 구현
- 7.0 상세 기능 설계 완료

## 8. 완료 판정 기준

6.9는 아래가 모두 충족되면 완료로 본다.

- 공유 동의문구가 owner 화면과 정책 문서에서 같은 의미로 정리된다.
- lawyer 화면의 열람·다운로드·출력 안내문이 실제 권한 모델과 일치한다.
- revoke, denied, log retention 관련 문구가 실제 API 결과와 모순되지 않는다.
- AI 생성 문서의 책임 한계와 변호사 최종 검토 책임이 분리되어 표현된다.
- 개인정보 제3자 제공/열람 동의 구조가 문구와 scope 개념 모두에서 정리된다.
- 전자소송 제출 보조 패키지의 책임 제한 문구 초안이 7.0로 넘길 수 있는 수준으로 정리된다.
