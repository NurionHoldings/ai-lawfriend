본 문서는 AI법친의 전체 화면을 **무조건 많이 만드는 순서**가 아니라,  
**MVP 주경로와 기준문서 잠금 결과를 실제 화면 우선순위에 반영**하기 위한 문서다.

즉, 어떤 화면이 먼저 안정화되어야 하고,  
어떤 화면은 후순위로 미뤄도 되는지를 고정한다.

---

## 1. 목적

이 문서의 목적은 다음과 같다.

1. MVP에 필요한 화면 우선순위를 고정한다.
2. 화면 개발 순서를 상태/권한/인터뷰/문서 흐름과 맞춘다.
3. 운영 고도화 화면과 핵심 사용자 흐름 화면을 구분한다.
4. 개발팀이 “무엇부터 마감해야 하는지”를 흔들리지 않게 한다.
5. 후속 API 명세와 화면 구현 역점검의 기준이 된다.

---

## 2. 기본 원칙

### 2-1. 화면 우선순위는 MVP 주경로를 따른다

다음 흐름을 최우선 기준으로 본다.

1. 변호사 가입/승인
2. 로그인
3. 대시보드
4. 의뢰인 초대/가입
5. 사건 생성
6. AI 인터뷰
7. 사건 요약
8. 문서 생성
9. 사건 상세
10. 보완 요청
11. 관리자 승인

### 2-2. 기준문서가 잠긴 흐름부터 화면을 정렬한다

- 상태값
- 라이프사이클
- 권한
- 질문셋
- 문서 템플릿  

기준이 잠긴 흐름부터 화면 우선순위를 부여한다.

### 2-3. 운영 고도화 화면은 후순위다

감사로그, 운영 대시보드, 고급 큐, 분석 위젯 등은 중요하지만,  
MVP 핵심 경로보다 앞서면 안 된다.

### 2-4. 같은 화면이라도 “필수 범위”와 “고도화 범위”를 나눈다

예:

- 사건 상세 화면
  - 필수: 상태, 인터뷰, 문서, 첨부, 타임라인
  - 고도화: 비교 패널, 운영 위젯, 고급 필터

## 3. 우선순위 등급 정의

| 등급 | 의미 |
|------|------|
| P0 | MVP 필수. 없으면 기본 흐름이 끊김 |
| P1 | MVP 직후 필수. 흐름은 이어지지만 품질/운영이 크게 떨어짐 |
| P2 | 중요하지만 후순위 가능 |
| P3 | 고도화/운영 최적화 단계 |

## 5. 화면별 상세 우선순위

### 5-1. 변호사 가입/승인 화면

#### 우선순위

- P0

#### 이유

- 시스템 첫 진입점
- 권한정의서와 직접 연결
- 승인되지 않은 실무자 계정은 핵심 기능을 수행할 수 없음

#### 필수 범위

- 회원가입 폼
- 역할 기반 신청
- 승인 대기 상태 표시
- 관리자 승인 연결

#### 고도화 범위

- 소속/전문분야 상세 프로필
- 승인 이력 추적

---

### 5-2. 로그인 화면

#### 우선순위

- P0

#### 이유

- 모든 기능 진입 전제

#### 필수 범위

- 이메일/비밀번호 로그인
- 기본 오류 처리
- 세션 유지
- 권한별 리다이렉트

#### 고도화 범위

- 2차 인증
- 로그인 이력

---

### 5-3. 대시보드

#### 우선순위

- P0

#### 이유

- 사건 목록 진입 허브
- 역할별 현재 작업 진입점

#### 필수 범위

- 내 사건 목록
- 상태별 필터
- 최근 작업
- 사건 생성 버튼

#### 고도화 범위

- KPI 위젯
- 운영 알림
- 성과 분석

---

### 5-4. 의뢰인 초대/가입 화면

#### 우선순위

- P0

#### 이유

- 사건의 실제 사용자 연결
- 질문셋/인터뷰 참여와 연결

#### 필수 범위

- 의뢰인 초대
- 사건 연결
- 가입 상태 확인

#### 고도화 범위

- 다중 의뢰인 관리
- 초대 재발송/만료 관리

---

### 5-5. 사건 생성 화면

#### 우선순위

- P0

#### 이유

- 사건 도메인의 시작점
- `CREATED` 상태 진입과 직접 연결

#### 필수 범위

- 사건 제목/유형
- 초기 개요
- 담당자 배정
- 기본 상태 생성

#### 고도화 범위

- 템플릿형 생성
- 사건 유형별 자동 필드 분기

---

### 5-6. AI 인터뷰 화면

#### 우선순위

- P0

#### 이유

- 질문셋 정의서와 직접 연결
- `IN_INTERVIEW` → `INTERVIEW_DONE` 핵심 구간

#### 필수 범위

- 질문 표시
- 답변 저장
- 필수 질문 체크
- 분기 로직 반영
- 인터뷰 완료 처리

#### 고도화 범위

- 중간 요약
- 자동 스크롤/탭 구조
- 음성/파일 업로드 강화

---

### 5-7. 사건 요약 화면/카드

#### 우선순위

- P0

#### 이유

- 인터뷰 결과를 문서 입력 구조로 바꾸는 중간 산출물
- 사건 상세와 문서 생성 사이 다리 역할

#### 필수 범위

- 사건 개요
- 사실관계 요약
- 현재 상태 요약
- 다음 단계 요약

#### 고도화 범위

- 내부용/외부용 전환
- 누락 정보 강조
- 요약 버전 비교

---

### 5-8. 문서 생성/초안 화면

#### 우선순위

- P0

#### 이유

- `DRAFTING` 구간 핵심 화면
- 문서 템플릿 정의서와 직접 연결

#### 필수 범위

- 템플릿 선택
- 문서 생성
- 문단 구조 표시
- 문단 재생성
- 초안 고지문 표시

#### 고도화 범위

- diff 보기
- 문단 비교 패널
- 복수 문단 일괄 재생성

---

### 5-9. 사건 상세 화면

#### 우선순위

- P0

#### 이유

- 전체 작업의 중심 화면
- 사건, 인터뷰, 요약, 문서, 첨부, 타임라인의 집합점

#### 필수 범위

- 사건 상태
- 기본 메타 정보
- 인터뷰 상태
- 사건 요약
- 문서 목록/진입
- 첨부 목록
- 타임라인

#### 고도화 범위

- 상태 배너
- 운영 위젯
- 관련 감사로그 패널
- 고급 비교 뷰

---

### 5-10. 보완 요청 화면/플로우

#### 우선순위

- P1

#### 이유

- `INTAKE_PENDING`, `REVIEW_PENDING` 등과 직접 연결
- 핵심이지만 사건 생성/인터뷰/문서 생성보다 한 단계 뒤

#### 필수 범위

- 보완 요청 생성
- 보완 사유 입력
- 보완 상태 반영

#### 고도화 범위

- 보완 요청 이력
- 알림 연동
- 자동 리마인드

---

### 5-11. 관리자 승인 화면

#### 우선순위

- P1

#### 이유

- 승인 구조 완성의 핵심
- `APPROVED`, `DELIVERED`, `CLOSED` 흐름 마감

#### 필수 범위

- 승인 대기 목록
- 승인/반려 처리
- 승인 메모
- 승인본 잠금

#### 고도화 범위

- 승인 diff 보기
- 검증코드 생성
- 승인 이력 상세

---

### 5-12. 질문셋 관리자

#### 우선순위

- P2

#### 이유

- 실제 운영에는 중요하지만, 초기 MVP에서는 기본 질문셋 고정값으로도 가능

#### 필수 범위

- 질문셋 목록
- 섹션/질문 편집
- 배포 상태 관리

#### 고도화 범위

- 조건 분기 시각화
- 버전 비교
- 권한별 편집 제한

---

### 5-13. 문서 템플릿 관리자

#### 우선순위

- P2

#### 이유

- 템플릿 운영에는 중요하지만, 초기에는 제한된 템플릿만으로도 가능

#### 필수 범위

- 템플릿 목록
- 섹션/문단 편집
- 템플릿 활성화/배포

#### 고도화 범위

- 템플릿 diff

- Add-On 정책 연결

---

#### 이유

#### 필수 범위

- 업로드

- 분류 코드 지정

## 8. 화면 우선순위와 API 명세 연결

화면 우선순위는 API 명세 1차본 작성 순서와도 연결된다.

### API 우선순위 묶음

- P0 화면용 API
  - 인증
  - 사건 생성/조회/수정
  - 인터뷰 시작/응답/완료
  - 사건 요약 생성/조회
  - 문서 생성/조회/문단 재생성

- P1 화면용 API
  - 보완 요청
  - 승인/반려
  - 전달/종결

- P2/P3 화면용 API
  - 질문셋 관리자
  - 템플릿 관리자
  - 첨부 분류
  - 검증
  - 감사/운영

## 9. 구현 대조 체크포인트

구현과 대조할 때는 아래를 본다.

- P0 화면이 실제로 주경로를 끊김 없이 잇는가
- 상태 전이가 화면 버튼과 맞는가
- 권한별로 접근 가능한 화면이 올바르게 잠겨 있는가
- 인터뷰 → 요약 → 문서 생성 흐름이 실제로 이어지는가
- 승인 전/후 화면 구분이 있는가
- 운영 화면이 MVP 필수 화면보다 앞서 복잡해지지 않았는가

---

## 10. 미확정 항목

아래는 후속 확정이 필요하다.

1. `USER`의 사건 생성 허용 여부
2. 의뢰인용 전용 대시보드 범위
3. 승인 화면과 사건 상세 화면의 역할 분리 수준
4. 질문셋/템플릿 관리자에 제한적 `LAWYER` 권한 허용 여부
5. 운영 대시보드 최소 범위
