# AI법친 7.x 로드맵 분리 메모

## 1. 분리 사유
AI법친 6.x 사건 패키지 기능군은 배포 전 검증을 완료했고, 현재 Smoke Test 실측 대기 상태다.

따라서 6.x에는 신규 기능을 더 추가하지 않는다.
이후 개선·확장 항목은 7.x 로드맵으로 분리한다.

## 2. 6.x에서 하지 않을 것
- 사건 패키지 공유 정책 추가 확장
- 관리자 강제 공유 변경 기능
- 관리자 대리 다운로드 기능
- 신규 DB 모델 추가
- 신규 권한 상태값 추가
- PDF 레이아웃 대규모 변경
- AI 요약/문서 생성 로직 확장
- 변호사 검토 메모 고도화
- 대시보드 신규 기능 추가

## 3. 7.x 후보 주제
7.x 후보는 아래처럼 분리한다.

| 후보 | 설명 | 비고 |
|---|---|---|
| 7.0 | 운영 모니터링 대시보드 | 6.x 실사용 로그 기반 |
| 7.1 | 변호사 검토 메모 고도화 | 최종 법률판단은 변호사 |
| 7.2 | 보완 요청 워크플로우 고도화 | 의뢰인 재입력 흐름 |
| 7.3 | 사건 패키지 통계/리포트 | 관리자/운영자용 |
| 7.4 | PDF 출력 디자인 고도화 | 6.12 안정화 후 |
| 7.5 | 고객 피드백 수집 모듈 | 운영 데이터 기반 |
| 7.6 | 알림/메일 연동 | 공유 만료/보완 요청 |
| 7.7 | 보안 감사 리포트 | 열람/다운로드 로그 기반 |

## 4. 7.x 착수 조건
7.x는 아래 조건 충족 후 착수한다.

- 6.x Smoke Test 14/14 PASS
- 6.x 최종 운영 잠금 완료
- 운영 중 장애 P0/P1 없음
- 개인정보/보안 노출 이슈 없음
- 사용자 피드백 1차 수집
- 7.x 번호체계/정의서 선행 작성

## 5. 최종 원칙
6.x는 안정화.
7.x는 분리 설계.
기능 확장은 Smoke Test 완료 후 별도 번호체계로만 진행한다.
