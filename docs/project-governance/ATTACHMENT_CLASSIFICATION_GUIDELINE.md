본 문서는 AI법친에서 사건에 연결되는 **첨부자료(attachment)** 를 어떤 기준으로 분류하고,  
어떤 자료를 어떤 화면·문서·요약·검토 흐름에서 어떻게 취급할지를 정의한다.

첨부자료는 단순 파일 업로드가 아니라 아래를 포함하는 **업무 자산**으로 본다.

- 자료 유형  
- 신뢰 수준  
- 노출 범위  
- 연결 대상(질문·요약·문서·문단)  
- 필수 여부  
- 외부 노출 가능 여부  
- 보존·삭제·비노출 정책  

---

## 1. 목적

이 문서의 목적은 다음과 같다.

1. 첨부자료 유형 분류 체계를 고정한다.  
2. 질문셋, 사건 요약, 문서 템플릿과 연결 기준을 만든다.  
3. 내부 검토용과 외부 노출용의 자료 취급 차이를 정한다.  
4. 필수 자료·선택 자료의 판정 기준을 정한다.  
5. DB·API·화면이 같은 분류 체계를 쓰게 한다.  

---

## 2. 기본 원칙

### 2-1. 첨부자료는 사건 정보의 일부다

첨부자료는 단순 참고 이미지나 파일이 아니라,  
사건 사실관계와 문서 생성의 근거가 될 수 있는 입력 자원이다.

### 2-2. 자료의 존재와 자료의 신뢰는 다르다

파일이 업로드되었다고 해서 곧바로 사실이 확정되는 것은 아니다.  
자료는 **존재 여부**, **유형**, **참고 가치**, **검토 상태**를 구분해 다룬다.

### 2-3. 내부용과 외부용은 다르게 취급한다

모든 첨부자료가 외부 노출 대상이 되는 것은 아니다.  
내부 검토용 자료와 외부 전달 가능한 자료를 구분한다.

### 2-4. 질문셋과 첨부자료는 연결 가능해야 한다

질문셋의 `file` 유형 질문 또는 `reference_list` 유형 질문과 첨부자료는 연결될 수 있어야 한다.

### 2-5. 문서 생성 시 원문 전체를 무분별하게 복제하지 않는다

AI 출력은 첨부자료를 참조할 수 있으나,  
원문 전체를 필요 없이 그대로 노출하거나 복제하지 않도록 제한해야 한다.

## 3. 첨부자료 분류 체계

첨부자료는 크게 아래 6개 분류로 나눈다.

1. 신원·기초 자료  
2. 사건 직접 증거 자료  
3. 참고·배경 자료  
4. 내부 작성·가공 자료  
5. 대외 전달·출력 자료  
6. 기타·미분류 자료  

---

## 4. 첨부자료 유형 정의

### 4-1. 신원·기초 자료

사건 당사자 또는 기본 식별과 관련된 자료.

**예**

- 신분증  
- 사업자등록증  
- 위임장  
- 계약 당사자 확인 자료  
- 주소·연락처 확인 자료  

**기본 특성**

- 민감정보 포함 가능성 높음  
- 외부 노출 주의 필요  
- 문서 생성 직접 인용보다는 확인용으로 쓰이는 경우 많음  

---

### 4-2. 사건 직접 증거 자료

사건의 사실관계를 직접 뒷받침하는 자료.

**예**

- 계약서  
- 문자·메신저 캡처  
- 이메일  
- 녹취록  
- 사진·영상  
- 입금 내역  
- 진단서·확인서  
- 공문·통지서  
- 판결문·결정문·수사서류 일부  

**기본 특성**

- 문서 초안과 사건 요약에 직접 영향 가능  
- 질문셋 답변과 연결될 수 있음  
- 원문 전체보다 핵심 내용 요약 반영이 우선일 수 있음  

---

### 4-3. 참고·배경 자료

사건 이해를 돕지만 직접 증거라고 단정하기 어려운 자료.

**예**

- 배경 설명 문서  
- 관련 기사  
- 참고 판례 정리본  
- 제3자 설명 자료  
- 외부 링크 정리본  

**기본 특성**

- 직접 증거보다 보조적 성격  
- 내부 검토용 가치가 큼  
- 외부 노출 필요성은 낮을 수 있음  

---

### 4-4. 내부 작성·가공 자료

시스템 또는 담당자가 내부적으로 만든 자료.

**예**

- 사건 요약 초안  
- 내부 검토 메모  
- 문서 초안  
- 문단 비교표  
- 재생성 이력 결과  
- 체크리스트  

**기본 특성**

- 외부 노출 제한 가능성 높음  
- 승인 전 자료는 특히 주의  
- 문서 버전 관리와 연결 가능  

---

### 4-5. 대외 전달·출력 자료

외부 또는 의뢰인에게 전달되는 결과물.

**예**

- 승인본 PDF  
- 전달용 사건 설명문  
- 전달 완료 문서  
- 검증코드 포함 출력본  

**기본 특성**

- 승인 상태와 연결  
- 외부 노출 허용 범위가 명확해야 함  
- 검증·전달 상태 관리 필요  

---

### 4-6. 기타·미분류 자료

즉시 분류하기 어려운 자료.

**예**

- 형식 불명 파일  
- 임시 업로드 파일  
- 사건과의 관련성 검토 전 자료  

**기본 특성**

- 임시 분류  
- 후속 검토 필요  
- 직접 출력·요약 반영 금지 가능  

---

## 5. 표준 분류 코드 (초안)

시스템·DB·API에서 아래와 같은 분류 코드를 사용할 수 있다.

| 코드 | 설명 |
|------|------|
| `IDENTITY` | 신원·기초 자료 |
| `EVIDENCE` | 사건 직접 증거 자료 |
| `REFERENCE` | 참고·배경 자료 |
| `INTERNAL_WORK` | 내부 작성·가공 자료 |
| `DELIVERABLE` | 대외 전달·출력 자료 |
| `UNCLASSIFIED` | 미분류 자료 |

실제 코드값은 구현·DB와 일치하도록 후속 정렬 필요.

## 7. 질문셋과의 연결

첨부자료는 [QUESTION_SET_DEFINITION.md](./QUESTION_SET_DEFINITION.md) 와 아래 방식으로 연결된다.

### 7-1. `file` 질문 유형

- 질문에서 직접 파일 업로드를 받는 방식  
- 예: 계약서 업로드, 증거 사진 업로드  

### 7-2. `reference_list` 질문 유형

- 이미 업로드된 자료를 목록으로 연결  
- 예: “참고할 증거를 선택해 주세요”  

### 7-3. 원칙

- `linkedQuestionKey` 를 통해 어떤 질문에서 들어온 자료인지 추적 가능해야 한다.  
- 필수 첨부 질문은 인터뷰 완료 조건 또는 문서 생성 조건과 연결될 수 있다.  
- 첨부자료 분류는 질문 유형만으로 확정하지 않고, 업로드 후 재분류 가능해야 한다.  

---

## 8. 사건 요약과의 연결

사건 요약은 첨부자료를 직접 또는 간접 참조할 수 있다.

### 허용 방식

- 증거 존재 여부 요약  
- 주요 자료의 종류 설명  
- 누락 자료 안내  
- 참고자료 보유 여부 표시  

### 제한 방식

- 민감한 원문 전체 직접 노출 금지  
- 내부용 첨부를 외부 요약에 그대로 노출 금지  
- 사실 확정처럼 보이는 표현 금지  

**예**

- 가능: “계약서 및 메시지 기록이 제출됨”  
- 주의: “계약 위반이 입증되었다”라고 단정 금지  

---

## 9. 문서 템플릿과의 연결

첨부자료는 문서 템플릿의 문단 생성 입력으로 연결될 수 있다.

### 예시

- 계약서 → 사실관계 문단  
- 메시지 캡처 → 경위 정리 문단  
- 판결문 → 참고 법적 맥락 문단  
- 진단서 → 손해·피해 설명 문단  

### 원칙

- 첨부자료 전체를 그대로 복제하기보다 필요한 핵심만 반영  
- 첨부 기반 생성 문단은 근거 자료 유형을 추적 가능하게 해야 한다  
- `linkedParagraphKey` 로 어떤 문단과 연결되는지 남길 수 있어야 한다  

---

## 10. 노출 범위 정책

첨부자료는 노출 범위를 아래처럼 관리한다.

| 구분 | 설명 |
|------|------|
| `internal_only` | 내부 담당자만 열람 가능 |
| `client_visible` | 의뢰인에게도 노출 가능 |
| `admin_only` | 관리자만 열람 가능 |
| `restricted` | 특별 권한자만 가능 |

### 원칙

- `INTERNAL_WORK` 는 기본적으로 `internal_only`  
- `DELIVERABLE` 은 원칙적으로 외부 노출 가능  
- `IDENTITY` 는 민감도에 따라 제한  
- `EVIDENCE` 도 사건 상황에 따라 외부 노출 제한 가능

## 11. 민감도 정책

첨부자료는 민감도에 따라 구분한다.

### 11-1. 일반

- 일반 안내문  
- 공개 가능한 참고자료  

### 11-2. 주의

- 계약서 일부  
- 연락처 포함 문서  
- 개인 진술 자료  

### 11-3. 민감

- 신분증  
- 의료자료  
- 수사·재판 관련 비공개 문서  
- 개인정보가 다량 포함된 자료  

### 원칙

- 민감 자료는 외부 노출 전 별도 확인 필요  
- 민감 자료 기반 요약·문서 생성 시 불필요한 정보 확산 금지  

---

## 12. 필수 자료 판정 기준

첨부자료는 사건 유형이나 질문셋에 따라 필수일 수 있다.

### 예시

- 계약 분쟁 → 계약서 필수 가능  
- 형사 고소 → 고소 관련 증빙 필수 가능  
- 행정 대응 → 처분서·통지서 필수 가능  

### 원칙

- 필수 여부는 질문셋·사건 유형·템플릿 요구조건과 연결  
- 필수 자료가 누락되면 인터뷰 완료 또는 문서 생성에 경고·차단 가능  

---

## 13. 처리 상태 정의 (초안)

첨부자료는 아래 상태를 가질 수 있다.

| 상태 | 설명 |
|------|------|
| `UPLOADED` | 업로드 완료 |
| `CLASSIFIED` | 분류 완료 |
| `LINKED` | 질문·요약·문서에 연결 완료 |
| `REVIEW_REQUIRED` | 검토 필요 |
| `EXCLUDED` | 출력·요약 대상에서 제외 |
| `ARCHIVED` | 보관 상태 |

실제 상태값은 DB·API와 맞춰 후속 조정 필요.

## 14. 제외·비노출 정책

아래 자료는 원칙적으로 직접 출력·요약 대상에서 제외될 수 있다.

- 미분류 자료  
- 내부 작성 중간 메모  
- 승인 전 내부 검토 산출물  
- 개인정보 과다 포함 자료  
- 사건 관련성 미확정 자료  

### 원칙

- “업로드됨”과 “출력에 사용 가능”은 다르다.  
- 제외 사유는 추적 가능해야 한다.

## 15. 삭제·보관 원칙

### 삭제

- 실제 물리 삭제 여부는 시스템 정책에 따름  
- 사건과의 연계 해제와 논리 삭제는 구분 가능  

### 보관

- 승인본·전달본과 연결된 자료는 보관 가치가 높음  
- 내부 검토용 중간 자료는 별도 보관 정책 가능  

### 원칙

- 삭제·보관은 [PERMISSION_DEFINITION.md](./PERMISSION_DEFINITION.md) 와 연결 필요  
- 전달 이후 자료 정리는 운영 정책과 맞물려야 함

## 17. 구현 대조 체크포인트

- 민감 자료가 외부용 화면에 직접 노출되지 않는가  
- 내부용·외부용 노출 범위가 구분되는가
