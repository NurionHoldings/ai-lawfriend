# AI법친 7.x 로드맵 마스터 플랜

## 1. 목적

AI법친 7.x는 6.x 사건 패키지 기능군의 최종 잠금 이후, 운영 데이터와 사용자 피드백을 바탕으로 별도 로드맵으로 분리 착수한다.

7.x는 6.x에 혼입하지 않는다.

## 2. 7.x 착수 조건

- 6.x 운영 배포 완료
- 6.x Smoke Test 14/14 PASS
- 6.x 최종 잠금 완료
- 운영 중 P0/P1 장애 없음
- 개인정보/보안 노출 이슈 없음
- 사용자 피드백 1차 수집
- 7.x 번호체계 선행 작성
- 7.x 정의서 선행 작성

## 3. 7.x 후보 범위

| 번호 | 작업명 | 목적 | 상태 |
|---|---|---|---|
| 7.0 | 운영 모니터링 대시보드 | 6.x 운영 상태 시각화 | 후보 |
| 7.1 | 변호사 검토 메모 고도화 | 사건 패키지 검토 품질 향상 | 후보 |
| 7.2 | 보완 요청 워크플로우 고도화 | 의뢰인 재입력 흐름 개선 | 후보 |
| 7.3 | 사건 패키지 통계/리포트 | 운영자/관리자 분석 | 후보 |
| 7.4 | PDF 출력 디자인 고도화 | 사건 패키지 출력물 품질 향상 | 후보 |
| 7.5 | 고객 피드백 수집 모듈 | 개선 요구 수집 | 후보 |
| 7.6 | 알림/메일 연동 | 공유 만료/보완 요청 알림 | 후보 |
| 7.7 | 보안 감사 리포트 | 열람/다운로드 로그 기반 감사 | 후보 |

## 4. 7.x에서 금지할 것

- 변호사를 대체하는 법률판단 자동화
- 승소/패소 단정
- 고소/소송 결과 예측 단정
- 위법행위 조언
- 증거인멸·수사회피 조언
- 의뢰인에게 최종 법률 판단을 AI가 직접 제공하는 구조
- 6.x 잠금 기능의 무단 변경

## 5. 7.x 우선순위 제안

1순위: 7.0 운영 모니터링 대시보드  
2순위: 7.5 고객 피드백 수집 모듈  
3순위: 7.7 보안 감사 리포트  
4순위: 7.2 보완 요청 워크플로우 고도화  
5순위: 7.1 변호사 검토 메모 고도화  
6순위: 7.4 PDF 출력 디자인 고도화  
7순위: 7.6 알림/메일 연동  

## 6. 7.x 첫 착수 권장안

7.x 첫 착수는 기능 확장보다 운영 안정성을 높이는 방향이 적합하다.

권장 시작:

AI법친 7.0 — 운영 모니터링 대시보드

이유:

- 6.x 배포 이후 실제 운영 상태를 추적할 수 있음
- 장애 대응과 보안 감사를 강화할 수 있음
- 기능 확장 전에 운영 데이터를 확보할 수 있음
- 관리자 관점의 플랫폼 신뢰도를 높일 수 있음

## 7. 최종 원칙

6.x는 잠금.
7.x는 분리.
운영 데이터 없이 기능 확장 금지.
법률 판단은 변호사 최종 검토 기준 유지.
