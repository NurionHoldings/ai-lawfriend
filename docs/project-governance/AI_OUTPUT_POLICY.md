본 문서는 AI법친에서 AI가 생성하거나 보조하는 출력물의 **허용 범위, 금지 범위, 승인 전·후 구분, 책임 경계, 노출 원칙**을 정의한다.

AI 출력은 단순 텍스트 생성 기능이 아니라,  
사건 인터뷰 답변과 문서 템플릿 구조를 바탕으로 한 **보조 산출물**로 본다.

---

## 1. 목적

이 문서의 목적은 다음과 같다.

1. AI가 생성 가능한 출력 범위를 정의한다.  
2. AI가 단독으로 확정하면 안 되는 영역을 정의한다.  
3. 승인 전·후 출력물의 차이를 고정한다.  
4. 문단 재생성, 사건 요약, 문서 초안의 정책을 통일한다.  
5. 사용자와 내부 담당자에게 어떤 문구와 수준으로 노출할지 기준을 만든다.  
6. 후속 **고지문·면책문구 정의서**의 상위 정책이 된다.  

---

## 2. 기본 원칙

### 2-1. AI는 보조자다

AI는 사건을 대신 판단하는 주체가 아니라,  
**입력된 사실과 구조를 정리하고 초안을 생성하는 보조자**다.

### 2-2. 확정 책임은 인간에게 있다

최종 판단, 승인, 전달, 종결 책임은 사람에게 있다.

- 실무 판단 책임: 담당 변호사·권한자  
- 운영 책임: 관리자·권한자  
- AI는 결정 보조 역할만 수행  

### 2-3. 승인 전과 승인 후는 다르다

- 승인 전 출력물: 초안  
- 승인 후 출력물: 승인본 또는 승인 기준 출력물  

이 둘은 같은 것으로 취급하지 않는다.

### 2-4. AI 출력은 입력 품질에 의존한다

질문셋 답변, 첨부자료, 상태값, 템플릿 구조가 부족하거나 불완전하면  
AI 출력의 품질도 제한된다.

---

## 3. AI 출력 대상

AI법친에서 AI 출력 대상으로 보는 것은 아래와 같다.

1. 인터뷰 답변 기반 사건 요약  
2. 문서 템플릿 기반 초안 생성  
3. 문단 단위 재생성  
4. 의뢰인 설명용 요약문  
5. 내부 검토용 구조화 문안  
6. 문서 작성 보조 문장 제안  

---

## 4. AI가 할 수 있는 것

아래는 AI가 허용 범위 안에서 수행 가능한 작업이다.

### 4-1. 구조화

- 장문 진술을 항목형으로 정리  
- 시간 순서 정리  
- 핵심 사실관계 요약  
- 질문 답변을 문단 구조에 맞게 재배열  

### 4-2. 초안 생성

- 사건 요약 초안 생성  
- 문서 템플릿 기반 문단 초안 생성  
- 결론·요청사항 초안 제안  
- 설명문·안내문 초안 제안  

### 4-3. 재생성·보조

- 특정 문단만 다시 생성  
- 더 짧게·더 명확하게·더 공식적으로 재작성  
- 누락된 입력을 기준으로 보완 질문 후보 제안  

### 4-4. 요약·병합

- 복수 질문 답변 병합  
- 첨부자료 설명과 진술 연결  
- 중복 내용 축약  

---

## 5. AI가 하면 안 되는 것

아래는 AI가 단독으로 확정하거나 사용자에게 사실처럼 단정해서는 안 되는 영역이다.

### 5-1. 법적 확정 표현

- “반드시 승소한다”  
- “법률적으로 확정되었다”  
- “최종 자문이 완료되었다”  
- “검토 없이 그대로 제출 가능하다”  

### 5-2. 책임 전가 표현

- “AI가 책임진다”  
- “자동 생성이므로 검토가 필요 없다”  

### 5-3. 승인 오인 표현

- 승인 전인데 승인 완료처럼 보이게 하는 표현  
- 내부 검토 중인데 확정본처럼 보이게 하는 표현  
- 전달 전인데 전달 완료처럼 보이게 하는 표현  

### 5-4. 근거 없는 사실 추가

- 입력되지 않은 사실을 임의로 보강  
- 첨부나 답변에 없는 내용을 추정 사실처럼 추가  
- 시간 순서를 임의 확정  

---

## 6. 출력 단계별 정책

### 6-1. 인터뷰 중 출력

**대상**

- 인터뷰 요약  
- 답변 정리  
- 추가 질문 제안  

**원칙**

- 참고용·진행 보조용으로만 표시  
- 사건 확정 의견처럼 보여서는 안 됨  
- 누락 정보가 있으면 명시적으로 표시  

### 6-2. 인터뷰 완료 후 출력

**대상**

- 사건 요약 초안  
- 문서 생성 입력 요약  

**원칙**

- 문서 생성 전 단계의 구조화 결과  
- 아직 승인 문서가 아님  
- 질문셋 답변 기준이라는 점이 분명해야 함  

### 6-3. 문서 초안 출력

**대상**

- 문단 초안  
- 전체 문서 초안  
- 재생성 문단  

**원칙**

- 승인 전 문서로 명확히 표시  
- 담당자 검토 전 확정 문서처럼 보이면 안 됨  
- 잠금되지 않은 문단은 변경 가능 상태로 본다  

### 6-4. 승인 후 출력

**대상**

- 승인본 기준 출력  
- 전달용 출력  
- 검증 가능한 결과물  

**원칙**

- 승인 여부가 명시되어야 함  
- 승인 시점 기준 구조를 보존해야 함  
- 승인 전 초안과 혼동되지 않게 구분해야 함

## 7. 상태값과 AI 출력의 연결

AI 출력은 사건 상태와 분리되지 않는다.

| 사건 상태 | AI 출력 원칙 |
|-----------|----------------|
| `CREATED` | 제한적 요약·접수 보조만 가능 |
| `INTAKE_PENDING` | 보완 안내·누락 정보 정리 중심 |
| `IN_INTERVIEW` | 인터뷰 보조, 중간 요약 가능 |
| `INTERVIEW_DONE` | 사건 요약 초안, 문서 입력 구조화 가능 |
| `DRAFTING` | 문서 초안 생성, 문단 재생성 가능 |
| `REVIEW_PENDING` | 검토용 출력, 승인 전 보정 가능 |
| `APPROVED` | 승인 기준 출력 가능 |
| `DELIVERED` | 전달 이후 기록성 출력 중심 |
| `CLOSED` | 신규 생성보다 조회·보관 중심 |
| `HOLD` | 신규 출력 제한 또는 예외 처리 |
| `REJECTED` | 재생성보다 종료·반려 맥락 중심 |
| `DELETED` | 일반 출력 대상 아님 |

---

## 8. 질문셋·템플릿과의 연결

### 8-1. 질문셋 입력 의존

AI 출력은 질문셋 답변을 기반으로 한다.

**원칙**

- 질문셋 입력 키 없는 출력은 최소화  
- 필수 질문 누락 시 출력 수준 제한  
- 분기 질문 결과에 따라 생성 문단도 달라져야 함  

### 8-2. 문서 템플릿 의존

AI 출력은 문서 템플릿 구조를 따른다.

**원칙**

- 템플릿 없는 자유 생성보다 템플릿 기반 생성을 우선  
- 문단 생성 방식(`ai_generate`, `ai_assist`, `manual_only`, `locked_reference`)을 따라야 함  
- 잠긴 문단과 고정 문구는 AI가 임의 변경하지 않음  

---

## 9. 문단 재생성 정책

문단 재생성은 허용하되, 아래 원칙을 따른다.

1. 재생성은 문단 단위로 한다.  
2. 재생성 전 원문 보존 근거가 있어야 한다.  
3. 잠긴 문단은 일반 재생성 대상이 아니다.  
4. 승인본 기준 문단은 별도 권한과 확인이 필요하다.  
5. 재생성 결과는 기존 문단과 비교 가능해야 한다.  

---

## 10. 사용자 노출 정책

### 10-1. 내부 담당자용

**허용**

- 사건 요약 초안  
- 문서 초안  
- 문단 재생성 결과  
- 보완 질문 제안  
- 검토용 비교 결과  

### 10-2. 의뢰인·외부 사용자용

허용 범위는 더 좁다.

**원칙**

- 내부 검토 중 문서 전체를 그대로 노출하지 않음  
- 설명용 요약은 가능하되, 법률 확정 의견처럼 보이면 안 됨  
- 승인 전 초안은 외부 노출 시 별도 고지 필요  

---

## 11. 승인 전·후 구분 정책

| 구분 | 승인 전 | 승인 후 |
|------|---------|---------|
| 성격 | 초안·보조 출력 | 승인 기준 산출물 |
| 수정 가능성 | 높음 | 제한적 |
| 재생성 가능성 | 가능 | 제한적·권한 필요 |
| 고지 필요성 | 매우 높음 | 승인 상태 표시 중심 |
| 전달 가능성 | 원칙적으로 제한 | 가능 |

### 원칙

- 승인 전 출력은 항상 초안성 표시가 필요하다.  
- 승인 후 출력은 승인 시점 기준의 구조와 상태를 보존해야 한다.  

---

## 12. 금지 표현 정책

AI 출력에는 아래 표현이 직접 들어가지 않도록 한다.

### 금지 표현 예시

- “최종 확정”  
- “검토 완료로 간주”  
- “법률상 보장”  
- “무조건 가능”  
- “AI 판단상 확정”  
- “검토 없이 제출 가능”  

### 처리 원칙

- 금지 표현 탐지 시 재생성 또는 검토 요청  
- 템플릿·정책 태그와 연결하여 자동 검사 가능성 검토  

---

## 13. 민감정보 처리 원칙

AI 출력은 민감정보를 다룰 수 있으므로 아래 원칙을 따른다.

1. 필요 없는 개인정보는 과도하게 노출하지 않는다.  
2. 외부 노출용 문서와 내부 검토용 문서를 구분한다.  
3. 첨부자료 내용을 반영할 때 원문 전체를 불필요하게 복제하지 않는다.  
4. 요약문에는 최소한의 정보만 노출한다.
