본 문서는 AI법친에서 사용자와 내부 담당자에게 노출되는 **고지문, 안내문, 면책문구**의 기준을 정의한다.

이 문서는 단순 문장 모음이 아니라 아래를 잠그기 위한 정책 문서다.

- 어느 화면·출력물에 어떤 문구를 붙일지  
- 승인 전과 승인 후에 어떤 표현 차이를 둘지  
- 내부용과 외부용의 문구 수준을 어떻게 구분할지  
- 사용자 오인 가능성을 어떤 문구로 차단할지  

---

## 1. 목적

이 문서의 목적은 다음과 같다.

1. 승인 전 초안이 확정 문서처럼 오인되지 않게 한다.  
2. AI 출력 결과가 법률 확정 의견처럼 보이지 않게 한다.  
3. 내부 검토용과 외부 노출용 문구를 구분한다.  
4. 승인 상태, 전달 상태, 검증 상태를 문구로 명확히 구분한다.  
5. UI·문서·API 응답에 붙는 고지문을 일관되게 유지한다.  

---

## 2. 기본 원칙

### 2-1. 고지문은 기능이 아니라 정책이다

고지문은 보기 좋게 붙이는 문장이 아니라,  
**오인 방지와 책임 경계 설정을 위한 정책 요소**다.

### 2-2. 승인 전 출력물은 반드시 초안성 표시가 필요하다

승인 전 문서, 문단 재생성 결과, 사건 요약 초안은  
항상 “확정 문서가 아님”을 분명하게 표시해야 한다.

### 2-3. AI는 보조자이며 최종 책임 주체가 아니다

AI 결과는 보조 초안으로 취급하며,  
최종 판단과 책임은 사람에게 있다.

### 2-4. 문구는 노출 대상에 따라 다르게 쓴다

- 내부 담당자용  
- 의뢰인·외부 사용자용  
- 승인본·비승인본  
- 검증 페이지용  

문구는 대상에 맞게 구분한다.

### 2-5. 과도한 면책문구보다 명확한 안내가 우선이다

문구는 법률적으로 과장되거나 사용자에게 위협적으로 보이지 않도록 하되,  
핵심 오인 가능성은 분명하게 차단해야 한다.

---

## 3. 문구 분류 체계

고지문·면책문구는 아래 5종으로 나눈다.

1. **인터뷰 단계 안내문**  
2. **사건 요약·설명용 고지문**  
3. **문서 초안 고지문**  
4. **승인본·출력본 상태 고지문**  
5. **검증 페이지 안내문**  

---

## 4. 인터뷰 단계 안내문

### 4-1. 인터뷰 시작 전 기본 안내문

**적용 위치**

- AI 인터뷰 시작 화면  
- 사건 인터뷰 진입 모달  
- 인터뷰 진행 전 동의·안내 영역  

**기본 문구(초안)**

> 본 인터뷰는 사건 내용을 정리하고 문서 작성을 돕기 위한 기초 정보 수집 단계입니다.  
> 입력하신 내용은 담당자의 검토와 후속 절차를 거쳐 활용될 수 있으며, 현재 단계에서 최종 법률 판단이 확정되는 것은 아닙니다.

### 4-2. 인터뷰 중 보조 안내문

**적용 위치**

- 질문 응답 화면 상단·하단  
- 중간 요약 영역  
- 추가 질문 제안 영역  

**기본 문구(초안)**

> 현재 표시되는 요약과 제안 문구는 입력 내용을 바탕으로 한 정리 결과이며, 검토 전 참고용입니다.

### 4-3. 인터뷰 완료 직전 안내문

**적용 위치**

- 인터뷰 완료 버튼 주변  
- 완료 직전 점검 모달  

**기본 문구(초안)**

> 인터뷰 완료는 정보 수집 단계의 종료를 의미합니다.  
> 이후 문서 초안 생성 및 검토 과정이 별도로 진행될 수 있습니다.

---

## 5. 사건 요약·설명용 고지문

### 5-1. 내부 담당자용 사건 요약

**적용 위치**

- 사건 요약 카드  
- 인터뷰 완료 후 요약 화면  
- 내부 검토용 요약 패널  

**기본 문구(초안)**

> 본 요약은 인터뷰 및 입력 자료를 바탕으로 자동 정리된 초안입니다.  
> 검토 전 참고용으로 사용하십시오.

### 5-2. 의뢰인·외부 사용자용 사건 설명문

**적용 위치**

- 의뢰인용 사건 설명 화면  
- 요약 공유용 문서  
- 안내 메시지 영역  

**기본 문구(초안)**

> 본 내용은 사건 진행 이해를 돕기 위한 설명용 정리이며, 최종 법률 의견서 또는 확정 판단과 동일하지 않을 수 있습니다.

### 5-3. 요약 생성 실패·입력 부족 시 안내문

**적용 위치**

- 요약 생성 실패 화면  
- 입력 부족 경고 배너  

**기본 문구(초안)**

> 현재 입력된 정보만으로는 충분한 요약을 생성하기 어렵습니다. 추가 정보 확인 후 다시 진행해 주십시오.

---

## 6. 문서 초안 고지문

### 6-1. 문서 초안 기본 고지문

**적용 위치**

- 문서 생성 직후 화면  
- 문서 상세 상단  
- 문단 편집 화면  
- 초안 미리보기  

**기본 문구(초안)**

> 본 문서는 AI 보조를 통해 생성된 초안입니다.  
> 담당자 검토 및 승인 전에는 확정 문서로 사용할 수 없습니다.

### 6-2. 문단 재생성 결과 고지문

**적용 위치**

- 문단 재생성 패널  
- diff·비교 화면  
- 재생성 이력 보기  

**기본 문구(초안)**

> 아래 내용은 선택 문단에 대해 다시 생성된 제안안입니다.  
> 기존 승인 내용 또는 검토 결과를 대체하기 전에 반드시 확인이 필요합니다.

### 6-3. 잠금된 문단 관련 안내문

**적용 위치**

- 잠금 문단 표시 영역  
- 잠금 해제 요청 영역  

**기본 문구(초안)**

> 이 문단은 검토 또는 승인 기준으로 잠겨 있습니다.  
> 일반 재생성 또는 수정 대상이 아닙니다.

---

## 7. 승인본·출력본 상태 고지문

### 7-1. 승인 전 출력물

**적용 위치**

- 초안 출력본  
- 미리보기 PDF  
- 내부 공유본  

**기본 문구(초안)**

> 본 출력물은 승인 전 초안입니다.  
> 최종 제출·전달용 문서로 사용하지 마십시오.

### 7-2. 승인 완료 후 출력물

**적용 위치**

- 승인본 PDF  
- 승인본 상세 화면  
- 전달 직전 출력 확인 화면  

**기본 문구(초안)**

> 본 문서는 승인 기준에 따라 생성된 출력본입니다.  
> 실제 전달 및 보관은 별도 운영 절차에 따릅니다.

### 7-3. 전달 완료 이후 상태 문구

**적용 위치**

- 전달 완료 문서 상태 영역  
- 사건 상세 상태 배너  

**기본 문구(초안)**

> 본 산출물은 전달 완료 처리된 기준본입니다.  
> 이후 수정 또는 재생성은 별도 권한과 절차가 필요할 수 있습니다.

## 8. 검증 페이지 안내문

### 8-1. 검증 페이지 기본 안내

**적용 위치**

- 검증코드 입력 화면  
- 검증 결과 상단  

**기본 문구(초안)**

> 이 페이지는 승인 기준 출력물의 진위 확인을 위한 안내용 검증 화면입니다.  
> 검증 결과는 등록된 승인 정보와의 일치 여부를 기준으로 표시됩니다.

### 8-2. 검증 성공 문구

**적용 위치**

- 검증 결과 성공 상태  

**기본 문구(초안)**

> 입력한 검증 정보와 일치하는 승인 기록이 확인되었습니다.

### 8-3. 검증 실패 문구

**적용 위치**

- 검증 결과 실패 상태  

**기본 문구(초안)**

> 입력한 검증 정보와 일치하는 승인 기록을 확인하지 못했습니다.  
> 코드 입력값 또는 문서 상태를 다시 확인해 주십시오.

## 9. 내부용·외부용 문구 구분

| 구분 | 내부 담당자용 | 외부·의뢰인용 |
|------|---------------|----------------|
| 초안 고지 | 더 직접적 표현 가능 | 더 부드럽고 설명형 문구 |
| 검토 필요성 | 명시적으로 표시 | 오인 방지 중심으로 표시 |
| 승인 전 경고 | 강하게 표시 | 혼동 방지 중심 |
| 시스템·운영 설명 | 일부 허용 | 최소화 |

### 원칙

- 내부용은 검토·승인·재생성 상태를 더 구체적으로 표시할 수 있다.  
- 외부용은 불필요한 내부 프로세스 표현을 줄이고, 결과 성격만 설명한다.  

---

## 10. 금지 표현 규칙

아래 표현은 승인 전 또는 AI 출력 영역에서 직접 사용하지 않는다.

### 금지 표현 예시

- “최종 확정되었습니다”  
- “법률적으로 보장됩니다”  
- “검토 없이 제출 가능합니다”  
- “AI 판단상 문제 없습니다”  
- “자동 승인되었습니다”  
- “확정 자문입니다”  

### 처리 원칙

- 금지 표현이 포함되면 수정 또는 재생성 대상으로 본다.  
- 후속 자동 검사 대상 후보로 관리할 수 있다.

## 11. 상태별 문구 노출 원칙

| 상태 | 문구 원칙 |
|------|-----------|
| `IN_INTERVIEW` | 수집·정리 단계임을 강조 |
| `INTERVIEW_DONE` | 문서 생성 전 구조화 완료 상태임을 강조 |
| `DRAFTING` | 초안 생성 단계임을 강조 |
| `REVIEW_PENDING` | 검토 전·검토 대기 상태임을 강조 |
| `APPROVED` | 내부 승인 완료 상태임을 강조 |
| `DELIVERED` | 전달 완료 상태임을 강조 |
| `CLOSED` | 종결 후 기록·보관 상태임을 강조 |

## 12. 권한과 문구 노출의 연결

### 원칙

- 같은 문서라도 역할에 따라 노출 문구 수준이 달라질 수 있다.  
- 내부 담당자에게는 검토·잠금·승인 상태를 더 구체적으로 표시한다.  
- `USER` 에게는 내부 승인·운영 용어를 과도하게 노출하지 않는다.  

### 예시

- `LAWYER`·`ADMIN`: “승인 전 초안”, “잠금 문단”, “재생성 이력” 표시 가능  
- `USER`: “설명용 정리”, “검토 전 참고용”, “최종 확정 전” 정도로 단순화  

---

## 13. 문서 템플릿과의 연결

문서 템플릿 정의서의 `generationMode`, `policyTags` 와 고지문은 연결 가능해야 한다.

**예**

- `locked_reference` 문단 → 고정 고지문 문단 후보  
- `policyTags: ["draft_notice"]` → 초안 고지문 삽입 대상  
- `policyTags: ["approval_notice"]` → 승인 상태 표시 대상  
- `policyTags: ["validation_notice"]` → 검증 페이지 안내 대상  

---

## 14. 운영상 판정 기준

고지문·면책문구가 적절하다고 보기 위한 조건은 아래와 같다.

1. 초안과 확정본이 문구로 구분된다.  
2. 승인 전 결과가 확정 의견처럼 보이지 않는다.  
3. 외부 사용자에게 내부 검토 문구가 과도하게 노출되지 않는다.  
4. 검증 페이지 문구가 과장되지 않는다.  
5. 금지 표현이 포함되지 않는다.  

---

## 15. 미확정 항목

아래는 후속 확정이 필요하다.

1. 승인 전 외부 공유 허용 시 문구 수위  
2. 의뢰인 설명문과 내부 요약문 간 문구 차등 수준  
3. PDF·출력본 워터마크와 문구의 중복 여부  
4. 자동 삽입 위치(상단 배너·본문·각주) 표준화  
5. 검증 실패 시 고객센터·재문의 안내 문구 추가 여부  

---

## 16. 구현 대조 체크포인트

구현과 대조할 때는 아래를 본다.

- 초안 화면에 고지문이 실제 표시되는가  
- 승인본과 비승인본의 문구가 구분되는가  
- 문단 재생성 결과에 안내문이 붙는가  
- 외부용 화면에서 내부 운영 문구가 과도하게 노출되지 않는가  
- 검증 페이지 성공·실패 문구가 준비되어 있는가
