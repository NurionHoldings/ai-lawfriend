**관련**

- 총괄 기준선: `docs/project-governance/TOP3_FINAL_INTEGRATED_JUDGMENT.md` §0-1

## 0. 이 문서의 목적

이 문서는 AI법친 공표 직전에 운영자가 마지막으로 확인해야 할 항목을 한 장으로 정리한 체크리스트다.

세부 판정은 각 개별 판정서와 증빙에 유지하고,  
이 문서는 **공표 직전 실제 운영 확인용**으로 사용한다.

---

## 1. 최종 운영 기준

- 세부 판정은 개별 문서에 유지한다.
- 총괄 기준선은 `TOP3_FINAL_INTEGRATED_JUDGMENT.md` §0-1 + `[262]`를 사용한다.
- 새 판정을 다시 여는 것이 아니라, 공표 직전 **운영·안내·실행 상태 확인**에 집중한다.
- 추가 발견은 재판정이 아니라 **선택 보강**으로 다룬다.

---

## 2. 공표 직전 최종 체크

### A. 기준 문서/증빙 고정

- [ ] `TOP3_FINAL_INTEGRATED_JUDGMENT.md` §0-1이 최신 상태다
- [ ] `IMPLEMENTATION_EVIDENCE.md` [262]가 최신 상태다
- [ ] 사건 상세 / 인터뷰→요약→문서 / 관리자 승인 각 축의 개별 판정서가 최신 상태다
- [ ] 공표 시 사용할 한 줄 기준 문장이 최종 확정되어 있다

### B. Top 3 핵심 축 상태

- [ ] 사건 상세 축은 [249] 기준으로 잠겨 있다
- [ ] 인터뷰→요약→문서 생성 축은 [254]~[256] 기준으로 잠겨 있다
- [ ] 관리자 승인 축은 [258]~[261] 기준으로 잠겨 있다
- [ ] Top 3 전체 종합은 [262] 기준으로 잠겨 있다

### C. 사용자 핵심 흐름 점검

- [ ] 사건 상세 진입이 정상 동작한다
- [ ] AI 인터뷰 진입/저장/완료가 정상 동작한다
- [ ] 사건 요약이 정상 생성/표시된다
- [ ] 문서 생성이 정상 동작한다
- [ ] 문서 승인/잠금/전달 준비가 정상 동작한다

### D. 권한/상태 차단 확인

- [ ] 비로그인 사용자는 보호 페이지 접근이 차단된다
- [ ] 역할별 버튼 노출이 맞다
- [ ] 인터뷰 완료 전 문서 생성이 차단된다
- [ ] 승인 전 잠금이 차단된다
- [ ] 잠금 전 전달 완료 처리가 열리지 않는다
- [ ] 삭제/종결/반려 등 금지 상태의 UI/라우트 차단이 맞다

### E. 운영 문구/안내 문구

- [ ] 상단 액션과 패널 액션의 역할 차이가 이해 가능하다
- [ ] 실패 메시지가 사용자 기준으로 이해 가능하다
- [ ] 승인 후 다음 행동 안내가 자연스럽다
- [ ] 잠금 후 검증/전달 안내가 자연스럽다
- [ ] 의뢰인/담당자 안내 문구가 혼동을 주지 않는다

### F. 데모/공표 시나리오

- [ ] 데모용 대표 사건 1개가 준비되어 있다
- [ ] 인터뷰 완료 상태 사례가 준비되어 있다
- [ ] 요약 생성 사례가 준비되어 있다
- [ ] 문서 생성 사례가 준비되어 있다
- [ ] 승인/잠금/전달 준비 사례가 준비되어 있다
- [ ] 발표 중 클릭 순서가 사전에 정리되어 있다

### G. 운영 추적/이력

- [ ] 타임라인에 승인/잠금 이력이 남는다
- [ ] 승인일/잠금일/버전 확인이 가능하다
- [ ] 검증 링크 동작을 확인했다
- [ ] 운영자가 후속 처리 순서를 이해하고 있다

## 3. 공표 가능 판정

### 공표 가능

- [ ] Top 3 축이 모두 잠겨 있다
- [ ] 핵심 사용자 흐름이 실제로 동작한다
- [ ] 실패/예외 메시지가 최소 운영 수준으로 이해 가능하다
- [ ] 데모 시나리오가 준비되어 있다

### 공표 전 미세 보정 필요

- [ ] 기능은 되지만 안내 문구가 거칠다
- [ ] 상단/패널 액션 의미가 조금 헷갈린다
- [ ] 일부 실패 메시지를 더 자연스럽게 바꿀 필요가 있다
- [ ] 데모 클릭 순서를 더 다듬을 필요가 있다

### 공표 보류

- [ ] 핵심 흐름 중 실제로 끊기는 항목이 있다
- [ ] 권한/상태 차단 오류가 있다
- [ ] 데모 중 오류 재현 가능성이 높다
