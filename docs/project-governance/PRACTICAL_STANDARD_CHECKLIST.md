# AI법친 실무용 표준 점검표

본 문서는 AI법친 프로젝트의 문서 잠금 상태와 화면 구현 상태를 에테르니언과 작업팀이 동일한 기준으로 점검하기 위한 운영용 체크리스트입니다.

사용 목적은 다음과 같습니다.

1. Phase 1~4 기준 문서의 현재 상태를 `존재`, `초안 사용 가능`, `잠금 완료`로 구분한다.
2. MVP 및 운영 핵심 화면 16개에 대해 구현·정합·검증 상태를 동일 형식으로 점검한다.
3. 새 기능 추가 전에 기준 문서 잠금과 구현 정렬 상태를 먼저 확인한다.
4. 증빙 번호와 실제 검증 명령을 함께 남겨, 이후 세션 재개 시 기준선으로 삼는다.

* **정합 확인 필요**: 기능은 있으나 권한/상태/문구/예외처리 검토 필요
* **운영 가능**: 기준선·권한·상태·에러 처리·증빙까지 포함해 실무 사용 가능

## 1-4. 잠금 후보 선별 (보수적)

**원칙:** `잠금 완료` 판정은 본 표에서 **아직 없음**(§1-3). 아래는 **향후 잠금 논의 시 우선 묶음**만 고른다. 단일 문서만으로 상태·권한 축을 잠그는 것은 **하지 않는다.**

| 구분 | 문서(들) | 선별 근거 |
| ---- | -------- | ---------- |
| **1순위 묶음 후보** | [CASE_STATUS_DEFINITION](./CASE_STATUS_DEFINITION.md) · [CASE_LIFECYCLE_DEFINITION](./CASE_LIFECYCLE_DEFINITION.md) · [PERMISSION_DEFINITION](./PERMISSION_DEFINITION.md) | 상태 의미·허용 전이·실행 주체가 **삼각 의존**. CASE_STATUS는 초안 1.3로 **상대적 성숙**이나 **코드 SSOT**·LIFECYCLE §11·PERMISSION §9 미확정이 남아 **단독 잠금 부적절**. |
| **단독 잠금 후보 아님** | 질문셋·문서 템플릿·AI 출력·NOTICE·사건 요약 명세 | 서로 및 Phase1 묶음과 **매핑 합의** 선행([108] 등). |
| **단독 잠금 후보 아님** | IO·첨부·DB 초안·화면 우선순위표·API 명세 | 구현·스키마·§22·[109]와 **동시 수렴** 필요. |

**요약:** 보수적으로 **「잠금 후보」= 위 Phase1 삼각 묶음 하나**로만 본다. 나머지 10개 문서는 **후속 묶음**으로 두고, 이번 점검표에서 단독 후보로 올리지 않는다.

### 1-4-1. 삼각 묶음(CASE_STATUS + LIFECYCLE + PERMISSION) — 잠금 논의 **전제 조건** (보수)

아래를 **모두** 충족할 때만, 운영상 **「삼각 묶음 잠금 완료」**를 **후보로 논의**할 수 있다. (본 점검표 §1-3의 **잠금 완료 0건** 판정은 **이 조건 미달**을 전제로 유지한다.)

1. **코드 SSOT 정렬:** `prisma/schema.prisma` 의 `CaseStatus` · `src/lib/definitions/case-status.ts` · [CASE_STATUS_DEFINITION.md](./CASE_STATUS_DEFINITION.md) **§2 목록·표시명**·§3(전이 위임)가 동일 선상에서 정합하거나, 불일치가 있으면 **코드 우선으로 문서가 갱신·증빙**되어 **남는 차이 0건**이다. (상태 토큰·라벨의 단일 목록은 §2 표.)
2. **라이프사이클 미확정 종료:** [CASE_LIFECYCLE_DEFINITION.md](./CASE_LIFECYCLE_DEFINITION.md) **§11** 항목이 **합의·문서 반영**되었거나, **명시적 차기 에픽**으로 분리되어 본 묶음 범위에서 **구현과 충돌하지 않음**이 표로 고정된다.
3. **권한 미확정 종료:** [PERMISSION_DEFINITION.md](./PERMISSION_DEFINITION.md) **§9** ①~⑥이 **각각 확정·본문 반영**되고, **§10** 체크포인트가 `middleware`·`assertCaseAccess`·관리자 라우트·상태 전이 API와 **대응표 + 샘플 증빙**(테스트 또는 점검 로그)을 갖는다.
4. **전이×역할 교차 검증:** 허용/금지 **전이**와 **실행 역할**이 세 문서 간 **상호참조 충돌 없이** 1:1 추적 가능하다(예: 전이 API·UI 버튼·금지 노출 **매트릭스** 1패스).
5. **역점검 1패스:** [ALIGNMENT_AUDIT_V1.md](./ALIGNMENT_AUDIT_V1.md) 또는 동등한 **정의서 대비 구현 역점검**에서, 상태·전이·권한 축이 **열린 이슈 없이** 닫히거나, 남은 항목이 **명시적 예외 목록**으로만 남는다.
6. **동시 확장 금지:** 본 묶음 잠금 논의는 **[108] 템플릿 게이트·[109] 감사/API 계약**의 **합의 전 확장**·`CaseStatus` **임의 스키마 변경**과 **같은 릴리스에 묶지 않는다** (충돌 시 잠금 보류).

**판정:** 위 1~6을 만족하지 못하면 **잠금 완료라고 부르지 않는다.** 「후보만 인정」(§1-4 표) 상태가 유지된다.

### 1-4-5. 전제 2 — LIFECYCLE **§11·전이 요약** 최소 산출물 형식·1차 표 구조

**전제:** `status_key` 축은 [CASE_STATUS_SSOT_MATRIX.md](./CASE_STATUS_SSOT_MATRIX.md) 와 동일한 **12토큰·동일 정렬**(`case-status.ts` 선언 순)을 쓴다.

#### 최소 산출물(파일)

- 본 저장소 [CASE_LIFECYCLE_TRANSITION_SUMMARY.md](./CASE_LIFECYCLE_TRANSITION_SUMMARY.md) 에 **전이 요약 표**를 둔다(마크다운 표 또는 동일 열의 CSV).
- 본문 [CASE_LIFECYCLE_DEFINITION.md](./CASE_LIFECYCLE_DEFINITION.md) §5~§8 등에 이미 있는 허용·금지 서술과 **충돌 없이** 같은 엣지는 동일 `kind` 로 적거나, 차이가 있으면 `열린 이슈` 행으로 남긴다.

#### 표 열(고정)

| 열 이름 | 내용 |
| ------- | ---- |
| `from_status` | 출발 `CaseStatus` (**12키 중 하나**). |
| `to_status` | 도착 `CaseStatus` (**12키 중 하나**). |
| `kind` | `ALLOW` (문서상 허용) / `DENY` (명시 금지) / `COND` (조건부·주석 필수) / `OPEN` (**§11·구현 미확정**). |
| `lifecycle_ref` | 근거 절(예: `LIFECYCLE §6-1`, `§11 bullet 3`). |
| `§11_item` | [CASE_LIFECYCLE_DEFINITION.md](./CASE_LIFECYCLE_DEFINITION.md) §11 **다섯 항** 중 해당하면 요약 키워드, 없으면 `—`. |
| `notes` | 한 줄 메모(조건·예외·에픽 ID). |
| `impl_ref` | (선택, B′) 구현 근거 경로·핸들러명; 미조사면 `—`. |

#### 1차 표 구조 — 행 배치 규칙

1. **희소 엣지 목록:** 12×12 전체를 채우지 않는다. **문서에 명시된 허용/금지/조건부 엣지**와 **§11에서 열린 엣지**만 행으로 둔다.
2. **정렬:** `from_status` 를 **SSOT 매트릭스와 같은 순서**로 그룹화하고, 각 그룹 안에서 `to_status` 를 **같은 순서**로 정렬한다.
3. **기본 해석(보수):** 표에 **없는** `(from,to)` 는 자동 허용으로 두지 않는다. 최종 허용 집합은 **LIFECYCLE 본문 §5~§8 + 본 요약표**를 합쳐 판단하며, 모순 시 `OPEN` 행을 연다.
4. **§11 전용:** §11 다섯 불릿은 처음에는 `kind=OPEN` 인 행(또는 행 묶음)으로만 올리고, 합의 후 `ALLOW`/`DENY`/`COND` 로 바꾼다.

**완료 판정(전제 2):** §11 각 불릿에 대해 `OPEN` 이 **0건**(또는 「차기 에픽·비포함」으로 **문서에 이관**)이고, 위 열이 채워진 요약표가 LIFECYCLE 본문과 **동기화**되면 §1-4-2 행 **2** 를 충족으로 올린다.
